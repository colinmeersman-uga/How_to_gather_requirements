# ChatGPT Prompts for Requirements Gathering

Use these prompts with transcripts, meeting notes, tickets, or stakeholder messages.

## Extract Requirements from a Transcript

```text
Analyze this requirements-gathering transcript. Extract user roles, user needs, user stories, acceptance criteria, edge cases, source-of-truth decisions, open questions, dependencies, story point considerations, and follow-up actions. Format the result for backlog refinement.
```

## Convert Notes into User Stories

```text
Convert these notes into Agile user stories using the format: As a [type of user], I want [an action] so that [a benefit]. For each story, add acceptance criteria in Given/When/Then format, priority, dependencies, and open questions.
```

## Improve Acceptance Criteria

```text
Review these user stories and acceptance criteria. Make the criteria more specific, testable, and complete. Include happy path, exception handling, audit/logging needs, and stakeholder validation criteria.
```

## Identify Edge Cases

```text
For this proposed workflow, identify edge cases, data quality risks, source-of-truth conflicts, failure modes, and operational support needs. Suggest acceptance criteria for each critical exception.
```

## Prepare for Backlog Refinement

```text
Prepare these requirements for backlog refinement. Group related stories, suggest priority, identify dependencies, estimate complexity drivers for story pointing, and list questions the team must answer before sprint planning.
```

## Summarize for Stakeholders

```text
Summarize these requirements for stakeholders in plain language. Include the business problem, desired outcome, user stories, acceptance criteria, open questions, and follow-up actions.
```
